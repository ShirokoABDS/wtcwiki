Wtcraft 服务器玩家使用文档

本页包含服务器规则、插件使用教程等内容，按模块分类整理。如仍有疑问，可在玩家群内询问，我们会持续完善本教程。

服务器规则 必读

本服为长期发展的纯生存养老社区服，支持轻度生电，注重社区氛围与原版生存体验。

欢迎来到 Wtcraft！进入服务器即视为同意以下全部规则，违反规则将视情节给予警告、禁言、封禁等处罚。

Wtcraft Wiki：https://shirokoabds.github.io/wtcwiki
  1. 账号安全与责任

    严禁盗号、骗号或利用他人账号违规操作。账号安全由持有人自负，所有操作视为持有人所为，违规将连带封禁关联账号。

  2. 文明交流与内容合规

    游戏内外（含群聊）禁止辱骂、人身攻击、引战、挂人、刷屏骚扰，也禁止发布违法不良信息、宣传其他服务器或商业广告。违者警告、禁言，严重者封禁或踢群。

  3. 财产与建筑保护

    严禁偷窃、恶意破坏、私自修改他人建筑、物资及驯养生物。私人借物纠纷管理不予受理，请自行确认风险。公共设施同样受保护，破坏者将按情节处罚。

  4. 领地保护期限（重要）

    所有重要建筑请务必使用圈地插件进行保护。自 2026 年 7 月 15 日 起，未圈领地的建筑若遭破坏，管理员将不再受理调查及补偿申请，请务必在此之前完成圈地。

  5. 生电与红石限制

    允许小型红石机械和常规农场，但禁止高频脉冲、长期高负载装置及任何明显影响服务器 TPS 的设施。管理员有权直接拆除违规机器，恶意卡服将从重处罚。

  6. 刷怪、挂机与小号管理

    仅允许单人小型刷怪塔，禁止集群高频刷怪装置。允许少量辅助小号，但禁止全自动挂机、公共区域挂机、多号切号等加重服务器负荷的行为。违规设备将被清理，主号连带处罚。

  7. 严禁外挂与作弊

    禁止使用任何第三方辅助工具（透视、飞行、脚本等）及利用 BUG 牟利。违者没收违规所得，并视情节封禁。举报有奖，恶意举报将反罚。

  8. PVP 与骚扰行为

    PVP 仅限指定区域，非 PVP 区禁止恶意攻击、陷阱杀害，禁止针对新玩家（装备差距过大视为恶意），禁止引怪围堵、卡位干扰他人。造成损失须全额赔付。

  9. 物资与交易限制

    普通鞘翅单人限持 2 个，仅可赠与不可交易，违规多持将递增处罚。其他物品交易请自行协商，管理不介入私人借贷纠纷。

  10. 环境与建筑规范

    禁止在出生点、新手区建造妨碍通行的设施。废弃工程请及时清理，避免浮空方块和杂乱建筑影响整体美观。违规建筑可直接拆除。

  11. 执法与申诉

    严禁阻挠管理员执法，违者加重处罚。对处罚有异议请私下向管理申诉，禁止在公共频道或群内当众扰乱秩序。

  12. 损失补偿受理条件

    因原版机制或自身失误导致的损失不予补偿。被偷窃或破坏需提供完整、无剪辑的证据链（含时间戳的截图或录屏）方可申请处理。

  13. 最终解释权

    管理团队保留本规则的最终解释权，对于未列明的违规行为，管理组有权根据实际情况酌情处理。违反规则将视情节给予警告、禁言、封禁等处罚，未尽事宜由管理组处理。

愿大家共同建设一个友好、稳定、长久发展的 Wtcraft 社区。

登录插件 账号安全

用于离线模式下的账号注册、登录、改密与邮箱找回。首次进服必须注册，之后每次上线需登录。

注册 / 登录

指令别名说明
/register <密码> <重复密码>/reg首次进服注册账号，两次密码需一致
/login <密码>/l每次上线登录账号

修改密码

指令别名说明
/changepassword <旧密码> <新密码> <重复新密码>/changepw修改登录密码，需提供旧密码

绑定邮箱（强烈建议）

绑定邮箱后，如忘记密码可直接通过邮箱接收验证码找回，无需联系管理员。

  1. 执行 /bindemail set 你的邮箱@example.com（或 /bdmail set），系统会向该邮箱发送验证码。
  2. 收到验证码后执行 /bindemail verify 验证码（或 /bdmail verify）完成绑定。

忘记密码（已绑定邮箱）

  1. 执行 /resetpassword forget（或 /repw forget），系统会再次向绑定邮箱发送验证码。
  2. 执行 /bindemail re 验证码 新密码（或 /bdmail re 验证码 新密码）设置新密码。
提示：密码请勿使用与其它平台相同的密码；建议进服后第一时间绑定邮箱，以免遗忘密码后无法找回。

皮肤插件 外观

支持直链皮肤、正版账号自动皮肤、LittleSkin 皮肤三种方式。

指令说明
/skin url "链接"使用皮肤直链（.png）设置皮肤，链接需用英文双引号包裹
/skin update更新当前皮肤（正版玩家换皮肤后同步）
/skin clear清除当前皮肤，恢复默认

三种获取皮肤方式

  1. 直链方式：已有皮肤 .png 直链时，使用 /skin url "https://example.com/skin.png" 即可。
  2. 正版账号：使用正版账号加入服务器，正常情况下服务器会自动为您应用对应正版账号的皮肤。
  3. LittleSkin 方式：前往 LittleSkin Getter 获取 littleskin 皮肤，或前往 SkinsRestorer Upload 上传，将网站给出的指令直接粘贴到服务器执行即可。
提示：直链需为可公开访问的 .png 图片地址（推荐 64×32 或 64×64 格式），切勿使用需要登录才能查看的链接。

BlockLocker 箱子锁 保护

通过在箱子上放置告示牌自动上锁，无需潜行。

  1. 手持告示牌，对着箱子（或其他容器）右键放置即可自动锁定，告示牌会显示你的ID。
  2. 只有告示牌上的玩家才能打开该容器。
重要安全提示：即便使用了告示牌箱子锁，仍强烈建议将箱子搬到家中并圈好领地。熊孩子会使用漏斗从底部吸走箱子中的物品；不圈领地还会导致基地被炸。箱子锁无法防漏斗，领地才是根本保护。

箱子商店 QuickShop-Hikari 交易

QuickShop-Hikari 是一款箱子商店插件，玩家无需任何指令即可通过箱子创建商店出售或收购物品，是最直观的玩家间交易方式

创建商店（最简方法）

整个创建过程不需要任何指令，只需三步：

  1. 放置箱子：在地上放一个箱子（或木桶等容器），作为商店的库存方块。
  2. 左键点击箱子：手持你要出售的物品，左键（不是右键）点击该箱子。聊天框会提示你输入物品单价。
  3. 输入单价：在聊天框输入物品单价（例如 10）并发送，商店即创建成功，箱子上方会出现悬浮展示物与告示牌显示交易信息。
易错点：创建商店必须用左键点击箱子，而不是右键。右键会直接打开箱子而不会触发商店创建流程，这是新手最常踩的坑。
提示：创建商店时手持的物品即为商店售卖的货种，物品的附魔、耐久、NBT 等数据都会被记录，后续补货必须放入完全一致的物品才能上架。

补货

商店创建好后，库存为 0 顾客无法购买。补货方式：

  1. 右键打开商店箱子（此时用右键才是打开）。
  2. 将与你商店货种完全一致的物品放入箱子。
  3. 关闭箱子，可售数量会随库存自动更新。

购买商品

作为顾客购买他人商店的商品：

  1. 找到目标商店，左键点击商店箱子或告示牌。
  2. 在聊天框输入你想购买的数量并发送。
  3. 系统自动扣款并将物品放入你的背包，交易完成。
提示：购买前请确认背包有足够空位与余额，背包空间不足可能导致交易失败或部分失败。

管理商店

商店创建后，可以通过两种方式管理：右键打开控制面板，或使用指令。

方式一：控制面板（推荐）

右键点击你的商店（或商店告示牌），会弹出聊天框控制面板，可点击对应文字一键完成以下操作：

  • 修改价格
  • 切换出售 / 收购模式
  • 冻结 / 解冻商店（临时歇业）
  • 查看交易记录
  • 移除商店
  • 管理店员
提示：控制面板是 QuickShop-Hikari 的核心交互方式，大部分日常管理都无需输入指令，点击即可完成。

方式二：指令管理

准星对准你的商店后，可使用以下指令（别名 /qs）：

指令说明
/qs price <价格>修改商店价格
/qs sell切换为出售模式（玩家向你购买）
/qs buy切换为收购模式（玩家向你出售物品）
/qs find <物品名>查找附近出售指定物品的商店（如 /qs find dia 查找钻石商店）
/qs staff add <玩家名>添加店员（可帮你管理库存）
/qs staff del <玩家名>移除店员
/qs staff list查看当前店员列表
/qs unlimited将商店设为无限库存（管理员专用）
/qs setowner <玩家名>更改商店所有者（管理员专用）
/qs clean清除所有库存为 0 的商店（管理员专用）
权限说明：表中最后三条标注为管理员专用，普通玩家执行无效。/qs unlimited 可将商店设为无限库存（适合官方物资店），/qs setowner 可转让商店所有权。

移除商店

拆除商店只需破坏商店箱子即可，商店会自动注销，箱内剩余物品会掉落。也可以通过右键控制面板中的「移除商店」选项操作。

安全提示：商店箱子本身没有锁，任何人都可能破坏你的商店箱子。请务必将商店建在已圈领地的区域内，关闭他人对该箱子的破坏权限，避免商店被恶意拆除或库存被盗。
收购模式注意：切换为收购模式后，其他玩家会把物品卖给你并从你账户扣款。请确保账户有足够余额，并定期清理收购箱中已满的物品，否则交易会失败。

网页商店 WebShopX 交易

基于网页的交易系统，玩家可在网页中进行官方物品购买、玩家物品购买/出售，以及拍卖。支持官方商城（B2C）+ 玩家市场（C2C）+ 拍卖行 + 订单履约一体化。

商店网址

请在浏览器中访问以下地址进入网页商店：

http://wtshop.fucku.top

首次使用（设置网页登录密码）

网页商店使用「Minecraft 用户名 + 网页密码」登录，密码需在游戏内首次设置：

  1. 进入服务器，在游戏内执行 /webshopx password 你的新密码（或 /ws password 你的新密码）。
  2. 打开浏览器访问商店网址，输入你的 游戏 ID 与刚才设置的 网页密码 即可登录。
  3. 忘记密码时，重复执行上述指令即可重置。
提示：网页密码与游戏登录密码是两套独立的密码，请勿混用；建议使用不同的密码以提升安全性。

游戏内指令

所有指令均可用别名 /ws 简写。例如 /ws help 等价于 /webshopx help。

指令说明
/webshopx help查看插件帮助
/webshopx password <新密码>设置 / 重置网页登录密码
/webshopx market [gui]打开市场 GUI（可带 gui 参数强制打开图形界面）
/webshopx market sell <price> [amount] [currency]兼容式快速上架：将手持物品以指定价格上架到玩家市场；amount 为数量，currency 为结算货币
/webshopx market logs [count]查看卖家近期成交日志（默认查看若干条，可指定条数）
/webshopx claim [all|ODR-...|MKT-...|CLM-...|MCL-...]领取待发货内容：不带参数领取全部；也可指定单个订单号领取（如 ODR-xxx）
/webshopx mailbox claim领取邮箱中待补偿的物品（自动发货失败时触发兜底）
/webshopx redeem create <shop> <game> [max] [perUserMax] [minutes] [code]创建兑换码（管理员专用）
/webshopx reload重载配置与网页资源（管理员专用）
权限说明：表中最后两条标注为管理员专用，普通玩家执行无效。兑换码创建与配置重载请联系管理组处理。

玩家操作流程

网页商店的核心使用闭环如下：

  1. 设置密码：游戏内执行 /ws password <密码>，首次使用必须先完成此步。
  2. 网页登录：浏览器打开商店网址，使用游戏 ID + 网页密码登录。
  3. 浏览与购买：在网页浏览官方商城与玩家市场，支持搜索、筛选、排序；选定商品后下单支付。
  4. 上架出售：在游戏内手持要出售的物品，执行 /ws market sell <价格> 快速上架；或在网页端进行更详细的上架操作（改价、备注、暂停/恢复/下架）。
  5. 参与拍卖：在网页对拍卖品出价，支持英式、荷兰式、维克里、烛台等多种拍卖算法，注意各拍卖模式的结束规则。
  6. 领取物品：下单或成交后，到游戏内执行 /ws claim 领取待发货内容；若自动发货失败，使用 /ws mailbox claim 从邮箱领取补偿物品。
  7. 查成交记录：执行 /ws market logs 查看自己作为卖家的近期成交日志。
重要提示：购买后请尽快执行 /ws claim 领取物品，避免长时间挂起导致订单异常；若领取失败，请第一时间执行 /ws mailbox claim 并保留订单号截图联系管理组。
交易安全：网页商店的所有交易均通过系统担保完成，请勿私下通过任何非官方渠道转账或扫码；管理员不会主动索要你的网页密码。

玩家传送 移动

向他人发起传送请求，需对方同意；也可拉黑玩家避免被打扰。

指令说明
/tpa <玩家名称>向该玩家发送「传送到他身边」的请求
/tphere <玩家名称>请求该玩家传送到你身边
/tpaccept接受传送请求（也可点击聊天框 [接受] 直接接受）
/tpdeny拒绝传送请求（也可点击聊天框 [拒绝] / [拒绝并拉黑]）
/denys add <玩家名称>拉黑指定玩家（不再收到其传送请求）
/denys remove <玩家名称>解除拉黑
提示：被频繁打扰时，可直接点击聊天框中的「拒绝并拉黑」一键处理。

Brewery 酿酒插件 趣味

Brewery 是一款替代原版酿造流程的插件，支持发酵、蒸馏、陈酿等完整酿酒流程，可制作真实能让玩家喝醉的酒精饮品。

酿造流程概览

酿造分为三个主要阶段，每个阶段都需要对应的设备：

  1. 发酵（炼药锅）：在炼药锅下方放置火源（火、岩浆、岩浆块或营火），加入水，再右键投入原料即可开始发酵。可用时钟右键炼药锅查看发酵时长。
  2. 蒸馏（酿造台）：将发酵后的饮品放入酿造台，使用烈焰粉作为燃料进行蒸馏，提升酒精度数。
  3. 陈酿（木桶）：将酒放入木桶中静置熟成，不同的木桶会影响饮品品质与口感。

常用指令

指令说明
/brew help查看插件帮助页面
/brew info查看自己的醉酒状态信息
/brew info <玩家名>查看指定玩家的醉酒状态（需对应权限）
/brew create <配方名>通过指令直接酿造（需管理员权限）

醉酒效果

饮用酒精饮品后会进入醉酒状态，表现为：步履蹒跚、聊天内容会被自动扭曲、出现各种状态效果；醉酒过度可能被踢出游戏或在 /home 位置醒来。低品质饮品还会带来宿醉等负面效果。

注意：不要在战斗、下界、高空或危险区域大量饮酒，醉酒失控可能导致死亡或掉落损失，服务器不受理此类补偿。

常见问题

Q1：忘记密码且未绑定邮箱怎么办？

A：请在玩家群内联系管理组，提供账号信息进行人工核验后重置。后续请尽快执行 /bindemail set 绑定邮箱。

Q2：皮肤设置后别人看不到？

A：让对方重新进入视野或执行 /skin update 刷新；若使用直链，确认链接为 .png 且可公开访问。

Q3：箱子锁了还是被偷？

A：箱子锁无法防止漏斗吸取。请将箱子搬到领地内，并圈好领地关闭容器权限。

Q4：传送请求没反应？

A：可能被对方拉黑（/denys）；或对方未注意聊天框提示，可提醒其点击 [接受]。

Q5：领地怎么圈？

A：详见 Dominion 官方玩家文档。

Q6：领地保护期限是什么意思？

A：自 2026 年 7 月 15 日起，未圈领地的建筑若遭破坏，管理员将不再受理调查及补偿申请，请务必在此之前完成圈地。

Q7：酿酒配方在哪里查？

A：配方在游戏中不会直接公开，需自行尝试原料组合。进阶内容（蒸馏、陈酿、自定义配方等）请前往 BreweryX 官方文档 查阅。

Q8：喝醉酒后掉进虚空/岩浆死了，能补偿吗？

A：不能。因醉酒失控导致的死亡属于自身行为导致的风险，不在补偿受理范围内。请避免在危险区域大量饮酒。

Q9：网页商店登录密码和游戏登录密码一样吗？

A：不一样。网页商店密码需在游戏内通过 /ws password <新密码> 单独设置，与游戏登录密码（/login）是两套独立系统。建议使用不同密码。

Q10：在网页买了东西，怎么拿到物品？

A：下单后到游戏内执行 /ws claim 领取待发货内容；如果自动发货失败，执行 /ws mailbox claim 从邮箱领取补偿物品。若仍失败，请保留订单号截图联系管理组。

Q11：怎么把自己的物品挂到网页商店卖？

A：在游戏内手持要出售的物品，执行 /ws market sell <价格> 即可快速上架；也可以登录网页端进行改价、备注、暂停/下架等更详细的管理。

Q12：网页商店打不开 / 登录失败怎么办？

A：请依次检查：① 网址是否为 http://wtshop.fucku.top；② 是否已在游戏内设置过网页密码（/ws password）；③ 输入的是游戏 ID 而非邮箱；④ 若密码遗忘，重新执行 /ws password <新密码> 重置后再登录。

Q13：箱子商店怎么创建？左键还是右键？

A：手持要卖的物品，左键（不是右键）点击箱子，然后在聊天框输入单价即可。右键只会打开箱子而不会触发商店创建。

Q14：箱子商店的箱子会被别人破坏吗？

A：会。QuickShop 商店箱子本身没有锁，任何人都能破坏。请务必将商店建在已圈领地的区域内，关闭他人对该箱子的破坏权限。

Q15：怎么改箱子商店的价格？

A：右键你的商店打开控制面板，点击「修改价格」即可；也可以准星对准商店后执行 /qs price <新价格>。

Q16：怎么把出售商店改成收购商店？

A：右键商店打开控制面板切换模式，或准星对准商店执行 /qs buy 切换为收购模式；用 /qs sell 切回出售模式。
